SVG Resolver

Introduction

The SVG standard is huge and there are many ways to write a svg data. One very useful feature is to define basic shapes info the defs part, and re-use these items many times, applying different transformation for a given element.

<?xml version='1.0' encoding='UTF-8'?>
<svg id="svg" width="100mm" height="200mm" viewBox="0 0 100 200" version="1.1"
  xmlns:xlink="http://www.w3.org/1999/xlink"
  xmlns="http://www.w3.org/2000/svg"
  xmlns:svg="http://www.w3.org/2000/svg">
  <defs id="defs">
    <circle id="my_circle" cx="0" cy="0" r="2.25" style="opacity:1;fill:#000080;stroke-width:0.2" />
    <rect id="my_rect" x="0" y="0" width="10" height="20" style="opacity:1;fill:#000080;stroke-width:0.2;" />
  </defs>
  <g id="layer1">
    <g id="layer2">
      <use href="#my_circle" transform="translate(10,10)" style="opacity:0.5" id="circle_left" />
      <use href="#my_circle" x="10" y="15"                                    id="circle_right" />
      <use href="#my_rect"   x="30" y="10"                id="rect_1" />
      <use href="#my_rect"   transform="translate(50,10)" id="rect_2" />
    </g>
  </g>
</svg>

But at the end of the day, an application (like PyCut) prefers to (or can only) work with a svg data where all elements (shapes) are defined without transformations, because evaluatings such transformations is not an easy task.

Fortunately, the package svgelements does the job...

This utility uses svgelements and write the result to a svg file (what svgelements does not). It also manage to preserve the ids of the resolved items as well as merging the styles of the referenced items with the used items.

Usage

From the command line

> python svgresolver.py test_basic.svg

A file named test_basic.resolved.svg is created

<?xml version='1.0' encoding='UTF-8'?>
<svg xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" xmlns:svg="http://www.w3.org/2000/svg" width="100mm" height="200mm" viewBox="0 0 100 200" version="1.1" id="svg">
  <g id="layer">
    <circle id="circle_left" cx="10.000" cy="10.000" r="2.250" style="opacity:0.5;fill:#080080;stroke-width:0.2"/>
    <circle id="circle_right" cx="10.000" cy="15.000" r="2.250" style="opacity:1;fill:#080080;stroke-width:0.2"/>
    <rect id="rect_1" width="10.000" height="20.000" x="30.000" y="10.000" rx="0.000" ry="0.000" style="opacity:1;fill:#800080;stroke-width:0.2;"/>
    <rect id="rect_2" width="10.000" height="20.000" x="50.000" y="10.000" rx="0.000" ry="0.000" style="opacity:1;fill:#800080;stroke-width:0.2;"/>
  </g>
</svg>

and PyCut can use this "resolved" svg data as input. Note how the ids are preserved. Also note how the styles are merged in the resolved items.
