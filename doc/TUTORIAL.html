PyCut Tutorial

Introduction

PyCut is a clone of jsCut, the excellent (at least for beginners) CAM, web-based programm, that takes as input svg files and generates G-Code. So why this rewrite ?

It begins as a "fun" project, in order to understand how a G-code generator works, and how easy (or hard) it is to achieve. Well, it is hard.

A second point is the major shortcoming of jsCut: the fact that it is a web application: sure for developpers who know javascript technology it is very productive, but how many users are able to improve jsCut on their own? only a few I guess: it is not as easy as one may think. This is due to my opinion to the fact javascript/htlm frameworks are just a pain to understand for people not used to the web technologies (which by the way change lets say every 3 years...). Traditional desktop applications are a lot easyier to hack (yes!)

Web technologies change every 3 years? An example is jsCut itself, which is based on the "polymer" HTML framework. Who which is new to javascript has ever heard of this lib before?! And "polymer" is also based on so-called "html imports", which are in 2021 theoretically no more allowed... Consider jsCut GUI itself : how many html files/lines of code are neccessary to achieve the task, compared to, when written in python, only a few Qt Ui files for the design and the python main program having max 1000 lines of code?

So PyCut is written in python, with the GUI with PySide6. Hopefully there will be users that will be able to contribute.

The excellent things from jsCut are (among others):

  • a great svg viewer, with an API to add svg path(s) to the viewer content -the gcode "paths" are the added path to the initial geometry-
  • a great gcode viewer -in webgl-
  • the c++ CAM stuff: hopefully a swig wrapper or something equivalent is feasible

So the idea is the following:

  • in PySide, display the WebGL stuff (QEngineView) DONE
  • in PySide, display the SVG viewer stuff (QGraphicsView with SvgItem) DONE (well, the op. Union/Interect etc. only)
  • Rest of the MainWindow in PySide DONE (nearly)
  • G-Code Generation with interface to the C++ CAM interface (lot of js code to understand) TODO

After that

  • Migrating WebGL stuff to "native" python (QGraphicsView I guess, or PyOpenGL ?) TODO

Nice to have:

  • G-Code generation with spindle control (actually absent in jsCut) actually in PySide GUI only, should not be too difficult to add to the generated G-Code
  • Similar feature as in LaserWeb-4 for SVG structure viewer / SVG viewer TODO
