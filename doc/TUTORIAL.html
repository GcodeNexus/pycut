PyCut Tutorial

Introduction

PyCut is a clone of jsCut, the excellent (at least for beginners) CAM, web-based programm, that takes as input svg files and generates G-Code. So why this rewrite ?

It begins as a "fun" project, in order to understand how a G-code generator works, and how easy (or hard) it is to achieve. Well, it is hard.

A second point is the major shortcoming of jsCut: the fact that it is a web application: sure for developpers who know javascript technology it is very productive, but how many users are able to improve jsCut on their own? only a few I guess: it is not as easy as one may think. This is due to my opinion to the fact javascript is just a pain to understand for people not used to the web technologies (which by the way change every 3 years...). Traditional desktop application are a lot easyier to hack (yes!)

An example is the GUI itself: how many html files/lines of code are neccessary to achieve the task, compared to, written in python, only a few Qt Ui files are needed for the design and the python main program with max 500 lines of code? Even the javascript GUI is written with the **polymer** library (as far as I have understood). Who has ever heard of this lib before?!

So pyCut is written in python, with the GUI with PySide. Hopefully there will be users that will be able to contribute.

The excellent things from jsCut are (if you did not realized it) among others:

  • svg viewer, with an API to add svg path(s) to the viewer content
  • job viewer - in wed-gl - unfortunately not in "standalone version" (need a webserver)
  • the c++ CAM stuff: hopefully a swig wrapper or something equivalent is feasible

Especially, the SVG viewer is great with the ability to svg paths to the view "on the fly"

The WebGL view is also great

So the idea is following

  • in PySide, display the WebGL stuff (QEngineView)
  • in PySide, display the SVG viewer stuff (QEngineView)
  • Rest of the MainWindow in PySide
  • G-Code Generation with interface to the javascript stuff

After that

  • Migrating WebGL stuff to "native" python (QGraphicsView I guess)
  • Migrating SVG Viewer stuff to "native" python (QSvgView I guess)
  • Migrating CAM stuff from javascript interface to python interface (c++ sources are there)

Nice to have would be:

  • Extension: G-Code generation with spindle control (actually absent in jsCut)
  • Similar feature as in LaserWeb-4 for SVG structure viewer / SVG viewer
