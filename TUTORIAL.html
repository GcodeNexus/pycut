PyCut Tutorial

Introduction

PyCut is a clone of jsCut, the excellent CAM, web-based programm, that takes as input svg files and generates G-Code. Why this rewrite ?

It begins as a "fun" project, in order to understand how a G-code generator works, and how easy (or hard) it is to achieve.

A second point is the major shortcoming of jsCut: the fact that it is a web application: sure for developpers who know javascript technology it is very productive, how many users are able to improve it on their own? only a few I guess: it is not as easy as one may think. This is due and to my opinion to the fact that it is written in javascript/webassembly. It is just a pain to understand it for people not used to web technologies (which change every 3 years...). Traditional desktop application are a lot easyier to hack (yes!)

An example is the GUI itself: how many html files/lines of code are neccessary to achieve the task, compared to, written in python, only 1 Qt Ui file is needed for the design and the python main program with max 500 lines of code?

So pyCut is written in python, with the GUI with PySide. Hopefully there will be users that will be abel to contribute.

The excellent things from jsCut are (if you did not realized it) among others:

  • svg viewer, with an API to add svg path to the viewer
  • svg structur viewer (unfortunately not yet in jsCut, but in LaserWeb-4) to select path items
  • job viewer - in wed-gl - unfortunately not in "standalone version" (need a webserver)
  • the c++ CAM stuff: hopefully a swig wrapper or something equivalent is feasible
